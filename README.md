# Share 共用模組
此目錄包含了可以被其他服務共用的核心模組。

## 目錄結構

```
share/
├── alert/         # 告警核心邏輯
├── databases/     # 資料庫操作封裝
├── label/         # 標籤管理
├── logger/        # 日誌處理
├── models/        # 資料模型
├── notify/        # 通知服務
└── parser/        # 解析器
```

## 模組說明

### Alert 告警模組
- 處理告警規則的檢查與觸發
- 支援絕對值(absolute)和變化幅度(amplitude)檢查
- 提供告警狀態管理和靜默期設置

### API 處理器
- 實現 RESTful API 的請求處理
- 包含認證、授權等中間件
- 統一的錯誤處理和響應格式

### Databases 資料庫模組
- 封裝 MySQL 操作
- 提供通用的 CRUD 接口
- 支援事務處理

### Label 標籤模組
- 提供標籤的 CRUD 操作
- 支援多租戶(realm)
- 標籤值支援一對多關係

### Logger 日誌模組
- 統一的日誌記錄接口
- 支援日誌輪轉和清理
- 可配置的日誌級別和格式

### Models 資料模型
- 定義所有資料結構
- 包含資料庫模型和 API 請求/響應結構
- 提供資料驗證方法

### Notify 通知模組
- 支援多種通知管道(Email、Teams、Webhook等)
- 提供通知重試機制
- 統一的通知配置管理

### Parser 解析器
- 支援多種數據源格式解析
- 提供統一的解析接口
- 可擴展的解析器架構

## 使用說明

1. 所有模組都應遵循依賴注入原則
2. 錯誤處理應使用統一的錯誤類型
3. 配置應通過環境變數或配置文件注入
4. 日誌記錄應使用統一的 logger 模組

## 開發規範

1. 所有公開的函數都應該有註釋說明
2. 錯誤訊息應該使用中文
3. 需要維護單元測試
4. 遵循 Go 標準項目結構
